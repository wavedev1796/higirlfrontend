# HG-F3-005 - Pantalla principal con branding, sesion persistente y ajustes de intereses

## Resumen

Se implemento una nueva experiencia post-login en el frontend de Hi Girl con una pantalla principal mas alineada al branding de la marca, persistencia de sesion al refrescar el navegador y un flujo mas discreto para la gestion de intereses.

## Objetivo

Mostrar una pantalla principal real despues de iniciar sesion, mantener al usuario dentro de su cuenta mientras exista un token valido y ocultar los intereses de la navegacion principal una vez que el onboarding se completa por primera vez.

## Alcance Implementado

- **Ruta raiz inteligente**: la pagina `/` ahora detecta si existe token y redirige a `/feed` o `/login` segun corresponda.
- **Login con persistencia real**: al detectar una sesion activa, la pantalla de login redirige automaticamente al feed o a la ruta solicitada por `callbackUrl`.
- **Proteccion de rutas**: `ProtectedRoute` usa `router.replace` para evitar volver al login con el boton atras y fuerza el onboarding cuando los intereses no alcanzan el minimo requerido.
- **Pantalla principal con branding**: el feed dejo de ser un placeholder y ahora funciona como home visual con hero, tarjetas, accesos rapidos y mensajes de marca.
- **Intereses ocultos del menu principal**: despues de completar el onboarding inicial, la navegacion principal ya no muestra intereses como acceso directo; la gestion queda centralizada en Ajustes.
- **Ajustes dedicados**: se agrego `/ajustes` para visualizar intereses guardados y volver a editarlos sin exponerlos en la home.

## Cambios Realizados

### 1. Ruta raiz con deteccion de sesion

Archivo:

```text
src/app/page.tsx
```

La ruta raiz ahora revisa si existe token en almacenamiento local y redirige a:

- `/feed` si la sesion sigue activa.
- `/login` si no hay token.

Esto evita que un usuario autenticado vuelva a la pagina publica al escribir `localhost:3000`.

---

### 2. Login que respeta sesiones activas

Archivos:

```text
src/features/auth/components/LoginView.tsx
src/features/auth/components/LoginForm.tsx
```

El login ahora:

- detecta si ya existe una sesion activa y redirige sin mostrar el formulario innecesariamente;
- respeta `callbackUrl` cuando el usuario fue enviado desde una ruta protegida;
- usa `router.replace` para evitar que el navegador vuelva al login despues del acceso.

---

### 3. Persistencia de autenticacion

Archivo:

```text
src/features/auth/providers/AuthProvider.tsx
```

La sesion sigue persistida en `localStorage` mediante `tokenStorage`, y al recargar el navegador el proveedor vuelve a hidratar el usuario consultando `/auth/perfil`.

Resultado funcional:

- si el token sigue vigente, el usuario permanece autenticado;
- si el token expira o ya no existe, el usuario cae de forma segura al flujo de login.

---

### 4. Navegacion protegida y onboarding

Archivo:

```text
src/features/auth/components/ProtectedRoute.tsx
```

La guardia de rutas ahora:

- redirige con `replace` a `/login` cuando no hay autenticacion;
- fuerza `/intereses` si el usuario no completa el minimo requerido para el onboarding;
- evita ciclos de navegacion innecesarios.

---

### 5. Home principal con identidad visual

Archivo:

```text
src/app/(dashboard)/feed/page.tsx
```

La pagina principal paso de ser un placeholder a una experiencia visual con:

- hero de bienvenida;
- tarjetas con métricas de contexto;
- accesos directos a Ajustes y Perfil;
- tratamiento visual basado en gradientes suaves y tarjetas tipo editorial.

---

### 6. Ajustes para intereses

Archivo:

```text
src/app/(dashboard)/ajustes/page.tsx
```

Se agrego una pantalla dedicada para revisar los intereses ya guardados y volver a editarlos desde un lugar mas natural para el usuario.

Esta pagina reemplaza el acceso directo permanente a intereses en la navegacion principal una vez que el onboarding ya fue completado.

---

### 7. Navegacion adaptada al estado del usuario

Archivo:

```text
src/app/(dashboard)/layout.tsx
```

El header del dashboard ahora cambia su menu segun el estado del usuario:

- si todavia no completa intereses, muestra el acceso correspondiente;
- si ya paso el onboarding, muestra `Ajustes` en vez de `Intereses`.

---

### 8. Rutas centralizadas

Archivo:

```text
src/shared/constants/routes.ts
```

Se agregaron rutas explicitas para:

- `INTERESTS`
- `SETTINGS`

Esto reduce el uso de strings sueltos en la navegacion del frontend.
