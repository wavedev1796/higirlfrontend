# HG-F3-003 - Pagina de Creacion de Cuenta

## Resumen

Se implemento la pagina `/register` para crear cuentas desde el frontend web de Hi Girl. La pantalla mantiene el fondo visual del login, centra el formulario, usa iconos por campo y redirige automaticamente al login cuando el registro es exitoso.

## Objetivo

Permitir que una nueva usuaria cree una cuenta desde la web usando el backend existente, sin incluir seleccion de roles en la interfaz.

## Ruta Implementada

```text
/register
```

## Endpoint Utilizado

```text
POST http://localhost:3000/api/v1/auth/register
```

## Payload Enviado

```json
{
  "firstName": "Magaly",
  "lastName": "Vinueza",
  "email": "magaly@test.com",
  "password": "Password123"
}
```

Nota: el frontend no envia `rol`. El backend asigna el rol por defecto configurado en el servicio/modelo.

## Alcance Implementado

- Nueva ruta `app/register/page.tsx`.
- Formulario centrado con diseno moderno.
- Fondo visual reutilizado del login.
- Eliminacion del bloque grande de texto del registro.
- Iconos SVG para identificar:
  - Nombre
  - Apellido
  - Correo electronico
  - Contrasena
  - Confirmacion de contrasena
- Validacion basica:
  - Nombre y apellido con minimo 2 caracteres.
  - Contrasena con minimo 8 caracteres.
  - Confirmacion de contrasena obligatoria.
  - Verificacion de que ambas contrasenas coincidan.
- Consumo de `POST /api/v1/auth/register`.
- Redireccion automatica a `/` despues de crear la cuenta.
- Link de regreso a login.

## Archivos Creados o Modificados

- `app/register/page.tsx`
- `app/page.tsx`
- `app/globals.css`

## Criterios de Aceptacion

- La ruta `/register` muestra el formulario de creacion de cuenta.
- El formulario esta centrado visualmente.
- Cada campo tiene un icono identificador.
- No existe selector de roles en la pantalla.
- El registro consume el endpoint real del backend.
- Al crear una cuenta exitosamente, la usuaria vuelve al login.
- El login tiene un link funcional hacia `/register`.
