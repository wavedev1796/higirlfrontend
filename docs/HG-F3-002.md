# HG-F3-002 - Integracion de Login con Backend

## Resumen

Se conecto el formulario de login del frontend con el backend NestJS de Hi Girl para validar credenciales reales mediante el endpoint de autenticacion.

## Objetivo

Permitir que la usuaria inicie sesion desde la web usando el backend existente y almacenar el token recibido para futuras rutas autenticadas.

## Endpoint Utilizado

```text
POST http://localhost:3000/api/v1/auth/login
```

## Payload Enviado

```json
{
  "email": "usuario@email.com",
  "password": "Password123"
}
```

## Respuesta Esperada

```json
{
  "mensaje": "Login correcto",
  "token": "...",
  "usuario": "...",
  "nombre": "...",
  "apellido": "...",
  "email": "...",
  "rol": "..."
}
```

## Alcance Implementado

- Envio del formulario mediante `fetch`.
- Uso de `NEXT_PUBLIC_API_URL` como variable configurable.
- Fallback local a `http://localhost:3000/api/v1`.
- Manejo de respuesta no exitosa con mensaje de error.
- Guardado del token en `localStorage` bajo la clave `higirl_token`.
- Presentacion de estado de sesion activa cuando el login responde correctamente.

## Criterios de Aceptacion

- El login consume `POST /api/v1/auth/login`.
- Si las credenciales son correctas, se almacena el token.
- Si las credenciales son incorrectas, se muestra un error.
- La UI no se bloquea permanentemente ante errores de red.
- El frontend funciona con el backend en `localhost:3000`.
